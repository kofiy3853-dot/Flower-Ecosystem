# Plan: Add Authentication Gating to Learning Features

## Problem

Server-side auth (`requireAuth` middleware) is solid, but the client-side is broken or missing. Unauthenticated users can view assignments, take quizzes, and see empty learning dashboards. The enrollment button is completely broken for all users.

## Auth Pattern to Follow

All pages should use this consistent pattern (already used correctly in `workshop-detail.html`, `garden-journal.html`):

```javascript
// At page init or before action:
if (!isLoggedIn()) {
    sessionStorage.setItem('pending-redirect', window.location.href);
    openAuthModal('login');
    return;
}
```

Functions come from `js/shared/auth.js`: `isLoggedIn()`, `openAuthModal()`, `getCurrentUser()`.
